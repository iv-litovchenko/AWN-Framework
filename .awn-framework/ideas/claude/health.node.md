# Health Check — проверка здоровья системы

## Когда запускать
Каждые **5 сессий**. Счётчик хранится внизу этого файла.

## Алгоритм

### 1. Реестр vs файлы
Сравнить список в `AGENTS.md` с реально существующими файлами.
- Файл есть, в реестре нет → добавить в реестр
- В реестре есть, файла нет → убрать из реестра

### 2. Дубликаты
Пройтись по node-файлам — нет ли двух нод которые описывают одно и то же.
- Если есть → объединить, лишний удалить

### 3. Мёртвые ссылки
Проверить все `[[wikilinks]]` в файлах.
- Ссылка есть, файла нет → создать файл или убрать ссылку

### 4. Memory
Открыть `memory.node.md` — актуально ли содержимое?
- Закрытые проекты → пометить как архивные
- Устаревшие факты → обновить или удалить

### 5. Мёртвые триггеры
Есть ли node которые давно не срабатывали?
- Триггер слишком узкий → расширить
- Нода потеряла смысл → архивировать

## После проверки
- Записать результат в `changelog.md`
- Обновить счётчик ниже
