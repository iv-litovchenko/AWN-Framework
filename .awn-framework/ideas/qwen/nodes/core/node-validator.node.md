# 🧹 Правило: Валидация узлов

## 🎯 Назначение
Гарантирует соблюдение принципов Digital SOLID при создании/изменении файлов.

## 📋 Чек-лист перед сохранением
Перед тем как создать или изменить `*.node.md`, проверь:

### 🔍 Уникальность
- [ ] Нет ли уже узла с такой же темой? (поиск по `TRIGGERS` и заголовкам)
- [ ] Если есть — лучше добавить ссылку `[[ ]]`, чем дублировать контент

### 📐 Структура YAML
- [ ] Все 5 обязательных полей заполнены: `TYPE`, `REQUIRE`, `PRIORITET`, `TRIGGERS`, `AUTOMATIZATION`
- [ ] `TYPE` — одно из: `guideline | skill | entity | rule | automation | template | memory`
- [ ] `REQUIRE` — только `always` или `on-demand`
- [ ] `PRIORITET` — `high | medium | low` или число 1-10
- [ ] `TRIGGERS` — массив ключевых слов в `[ ]`
- [ ] `AUTOMATIZATION` — только `true` или `false`

### 🗂 Реестр
- [ ] Если создан новый узел → добавлена строка в таблицу `AGENTS.md`
- [ ] Если удалён/переименован → таблица актуализирована

### 🧼 Чистота контента
- [ ] Нет повторяющихся абзацев внутри файла
- [ ] Нет копий текста из других узлов (только ссылки)
- [ ] Тело файла начинается с заголовка `#` после фронтматтера

## 🔗 Связи
- [[AGENTS.md]] — главный реестр
- [_template.node.md]] — шаблон для создания
