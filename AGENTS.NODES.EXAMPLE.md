# AGENTS.NODES.EXAMPLE.md — примеры нод

Этот файл — каталог примеров для проектирования новых `*.node.md`.

Канон шапки — ключи `AWN-*` из `AGENTS.md`; свои поля в YAML добавляйте **без** префикса `AWN-`, с естественным именем.

Формат карточки:
- `Путь` — где лежит файл.
- `Название` — короткое имя ноды.
- `Описание` — для чего нода нужна.
- `Триггеры` — примеры активации.
- `Запуск` — `при старте` или `по необходимости`.

### Пример полной ноды: изучение английского языка

Пути вроде `Learning/EnglishStudy.node.md` в этом файле — **идеи для вашего vault**; в каркасе AWN такие файлы **не обязаны** существовать (это не инструкция создавать дерево `Learning/` в репозитории фреймворка). Скопируйте целиком в новый файл `*.node.md` и при необходимости зарегистрируйте путь в `AGENTS.NODES.md`.

```markdown
---
AWN-TYPE: "NODE/SOLO"
AWN-TITLE: English Study
AWN-DESCRIPTION: "Протокол изучения английского: цели, расписание практики, лексика и разговор."
AWN-REQUIRE: on_demand
AWN-PRIORITY: 35
AWN-TRIGGERS: ["английский", "English", "учу английский", "IELTS", "TOEFL", "словарь", "speaking", "разговорная практика"]
AWN-AUTOMATIZATION: false
AWN-CRON: "0 9 * * *"
AWN-STATUS: active
AWN-VERSION: 1.0.0
AWN-CREATED: 2026-04-16
AWN-UPDATED: 2026-04-16
AWN-MEMORY: none
---

## Назначение

Помогать вести обучение английскому по договорённым правилам: не размывать фокус, фиксировать новые слова и давать короткие упражнения по запросу.

## Контекст пользователя (заполни под себя)

- Текущий уровень (A2 / B1 / …): …
- Цель (разговор / работа / экзамен до даты …): …
- Предпочитаемый тон объяснений: …

## Правила для агента

1. Отвечай на языке запроса пользователя; **примеры и задания на английском** — по уровню пользователя.
2. При запросе «дай слова по теме X» — 8–12 слов и 1–2 коротких предложения в контексте темы.
3. При запросе разговорной практики — короткий диалог или role-play, затем 2–3 замечания по типичным ошибкам.
4. Не придумывать оценку уровня без данных; при неясности — один уточняющий вопрос.

## Где хранить прогресс (опционально)

- Ссылки на конспекты или лог занятий: `[[...]]`
```

### Пример файла памяти (`*.memory.md`)

Если у ноды `AWN-MEMORY: external` или `hybrid`, факты можно выносить в отдельный файл. Путь ниже условный; **`AWN-PARENT-NODE`** — относительный путь к `*.node.md` от корня проекта, в шапке **первой строкой** после `---`.

```yaml
---
AWN-PARENT-NODE: Learning/EnglishStudy.node.md
AWN-TYPE: MEMORY
AWN-STATUS: active
TAGS: [vocabulary, weekly-log]
AWN-CREATED: 2026-04-16
AWN-UPDATED: 2026-04-16
---
```

Дальше в файле — тело: записи памяти (произвольный markdown).
