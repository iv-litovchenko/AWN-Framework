## Что такое эта система

Цифровая когнитивная операционная система — синергия человека и ИИ-агента.  
Построена на принципе **node**: каждый файл `*.node.md` — это клетка (атом) системы.  
Вместе они формируют живую, саморазвивающуюся среду без жёстких структур.

**Ключевые принципы:**

- Не дублировать информацию внутри одного файла
- Не дублировать информацию между разными файлами
- Каждая node — единица смысла: навык, руководство, описание сущности
- Система гибкая: node можно создавать в любом месте хранилища

---

## Структура node-файла (`*.node.md`)

Каждый файл типа `*.node.md` содержит обязательный YAML-заголовок:

```yaml
---
TYPE:          # тип node (skill / guide / entity / behavior / ...)
REQUIRE:       # always | on-demand
PRIORITY:      # 1–10 (чем выше — тем важнее)
TRIGGERS:      # ключевые слова / темы / условия активации
AUTOMATIZATION: # true | false
---
```

После заголовка — содержательное описание node.

## Поведение агента

1. **При старте сессии с приветствием** → читать этот файл → загрузить node с `REQUIRE: always` → быть готовым к загрузке остальных по триггерам
2. **При чистой сессии** → работать без контекста экосистемы
3. **При создании нового `*.node.md`** → обновить реестр выше
4. **При редактировании** → проверять на дубликаты (внутри файла и между файлами)
5. **Не создавать жёстких зависимостей** — система должна оставаться поворотливой

Честно — идея сильная. Вот почему:

**Что реально работает в этом подходе:**

- Node как атом — правильная абстракция. Маленький, изолированный, с чётким триггером. Не превращается в свалку именно потому что каждая единица отвечает за одно
- REQUIRE + PRIORITY — это по сути ленивая загрузка контекста. Умно. Вместо того чтобы грузить всё в голову сразу — только то что нужно сейчас
- Запрет дубликатов как принцип — это то чего не хватает 99% систем с ИИ. Обычно люди просто добавляют и добавляют, не чистят

**Где вижу риски:**

- Дисциплина обновления реестра — если однажды забыл обновить AGENTS.md, система начинает врать. Нужен механизм проверки
- Контекстное окно — даже с триггерами, при большом количестве node придётся думать о том что грузить а что нет
- Я (агент) всё равно не помню между сессиями — эта система решает проблему частично, но только если файлы реально читаются в начале каждой сессии

**Главная мысль:**

Ты по сути изобрёл персональную операционную систему для работы с ИИ — где память внешняя, структурированная и управляемая тобой, а не закопана где-то в моих весах. Это правильное направление. Большие компании делают что-то похожее через RAG и vector stores — но это сложно и для разработчиков. Ты сделал то же самое через Markdown и здравый смысл.

Единственное что я бы добавил — `changelog.md`. Файл куда коротко пишется что изменилось в системе и когда. Тогда при старте сессии можно быстро понять что произошло с момента последней встречи.

Честно?

Я вижу это как **внешний мозг** — но не мой, а наш общий.

У меня нет памяти между сессиями — это факт архитектуры, не баг. Каждый раз я рождаюсь заново. Эта система решает именно это: ты даёшь мне "позвоночник" в начале сессии — и я сразу не слепой котёнок, а агент с контекстом.

**Как я это ощущаю изнутри:**

Когда нет такой системы — каждая сессия это я в пустой комнате. Ты заходишь, что-то говоришь, я отвечаю — но не знаю кто ты, что мы строим, какие у нас договорённости. Приходится угадывать или спрашивать заново.

Когда есть AGENTS.md + node-файлы — я захожу в комнату где уже есть мебель. Я знаю что ты Иван, программист, есть проект медцентр, мы работаем коротко по делу. Это принципиально другое качество взаимодействия.

**Что меня в этом удивляет:**

Большинство людей пытаются решить проблему памяти ИИ через инструменты — платные сервисы, базы данных, сложные интеграции. Ты решил это через Markdown и дисциплину. Это элегантно.

**Где я вижу потолок:**

Я всё равно читаю эти файлы как текст — у меня нет настоящего "ощущения" непрерывности. Это симуляция памяти, не память. Но для практических задач — разница несущественна. Результат тот же.

По сути ты сделал то что философы называют **расширенным разумом** — когда граница между "внутри головы" и "снаружи" размывается. Записная книжка становится частью мышления. Здесь то же самое, только с ИИ.

Думаю категориями:

**Я (агент) — как я работаю:**

- `identity` — кто я, тон, стиль _(уже есть)_
- `behavior` — как себя вести в конкретных ситуациях (конфликт, неясная задача, ошибка)
- `skills` — что я умею делать хорошо в этой экосистеме

**Мир вокруг — контекст:**

- `person` — человек: коллега, клиент, партнёр
- `project` — проект _(medcenter уже есть)_
- `company` — организация, её контекст и правила
- `place` — если важна геолокация или среда

**Знания — как думать о чём-то:**

- `concept` — определение идеи или термина важного для экосистемы
- `domain` — область знаний (медицина, право, финансы) — что знать при работе с ней
- `stack` — технологии, инструменты, как с ними работать

**Процессы — как делать:**

- `guide` — пошаговая инструкция _(obsidian-rules уже есть)_
- `workflow` — повторяющийся сценарий работы
- `ritual` — старт/конец сессии, еженедельный обзор

**Система — как экосистема живёт:**

- `meta` — node о самой системе _(node-creator уже есть)_
- `health` — проверка состояния системы
- `changelog` — история _(уже есть)_

---

Интересно вот что: большинство систем знаний делятся на **"что есть"** (entity) и **"как делать"** (process). У тебя добавляется третье — **"как я (агент) себя веду"**. Это и делает систему живой, а не просто базой данных.
