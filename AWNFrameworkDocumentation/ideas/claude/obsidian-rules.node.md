# Правила работы с Obsidian-хранилищем

## Нейминг файлов
- `kebab-case` для всех файлов: `my-project.md`, `meeting-notes.md`
- node-файлы: `название.node.md`
- Без пробелов, без кириллицы в именах файлов

## Структура
- Нет жёстких папок — структура растёт органически
- Папка появляется только когда в ней больше одного файла
- Не создавать папку ради одного файла

## Линковка
- Связывать файлы через `[[wikilinks]]` — это нервная система экосистемы
- Если упоминаешь концепцию которая есть в другом файле — ставь ссылку

## Дубликаты
- Перед созданием нового файла — проверить не описано ли это уже
- Перед добавлением информации в файл — проверить нет ли её уже там
- Лучше расширить существующий node чем создать похожий новый

## Обновление AGENTS.md
- При создании любого `*.node.md` — немедленно добавить строку в реестр
- При удалении — убрать строку из реестра
