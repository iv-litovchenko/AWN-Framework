# Structure — Принципы организации хранилища

**Главный принцип:**  
**Гибкость важнее порядка.** Структура должна служить нам, а не мы — структуре.

**Правила (мягкие):**

- `*.node.md` — специальные файлы, которые агент читает автоматически. Используем только когда нужно дать мне новое поведение или знание.
- Всё остальное — обычные `.md` файлы.
- Папки создаём только когда это реально упрощает жизнь (не раньше).
- Дублирование запрещено. Лучше сделать хорошую MOC-страницу, чем копировать информацию.
- Если сомневаешься, где создать файл — клади в корень или в `Resources/`, потом можно переместить.

**Философия:**
Мы строим живую систему, а не библиотеку. Со временем структура будет меняться естественным образом. Это нормально.

**Моё поведение:**
Я буду предлагать оптимальное место для каждого нового файла и помогать поддерживать чистоту, но без фанатизма.
