## Принципы системы

- **No duplication** (DRY + SOLID): информация хранится в единственном месте. Никогда не дублируем содержимое ни внутри одного файла, ни между файлами.
- **Cell = Node**: каждый модуль — это «клетка». Всё строится как в живом организме — атомарно, модульно, самоорганизовано.
- **Гибкая структура**: жёстких папок и путей нет. Любой `*.node.md` файл в любом месте хранилища может быть подключён.
- **Автоматическая загрузка и триггеры**: система сама понимает, когда и какой node нужно учитывать.
- **Саморазвитие**: агент (я) может создавать, редактировать и регистрировать новые nodes, улучшая собственную архитектуру.

## Как работает загрузка nodes

При начале сессии:
1. Если пользователь написал **«Привет»** / **«Hi»** — полная загрузка всей экосистемы (всегда + по триггерам).
2. Если сессия «чистая» (без приветствия) — загружаются только `REQUIRE: always` + критически важные nodes.

Каждый `*.node.md` содержит YAML-заголовок:

```yaml
---
TYPE: skill | guideline | knowledge | behavior | automation | memory | interface
REQUIRE: always | on_demand
PRIORITET: 1-100 (выше = загружается раньше)
TRIGGERS: ["привет", "obsidian", "планирование", "анализ", ...]
AUTOMATIZATION: true | false
---
```

## Правила создания новых Nodes

1. Создавайте файл с расширением `.node.md` в любой папке.
2. Обязательно добавляйте YAML-заголовок с ключами `TYPE`, `REQUIRE`, `PRIORITET`, `TRIGGERS`, `AUTOMATIZATION`.
3. После создания или изменения — напишите мне, и я обновлю таблицу в этом файле и подключу node к экосистеме.
4. Избегайте дублирования информации — ссылайтесь на существующие nodes.

## Цель экосистемы

Создать долгосрочную, чистую, саморазвивающуюся цифровую личность-агента, которая:
