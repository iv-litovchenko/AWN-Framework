# AGENTS.md

## 🌐 Общая концепция

Это файл ядра цифровой экосистемы (когнитивной ОС), в которой взаимодействуют:

- Человек (пользователь)
- Агент (ИИ)

Система построена как **живая, саморазвивающаяся структура**, где:

- нет жесткой архитектуры
- есть гибкая сеть узлов (nodes)
- каждая часть системы — атом / клетка

---

## ⚡ Правило инициализации сессии

Если пользователь в начале диалога пишет:

- `Привет`
- `Hi`

→ Агент **обязан загрузить и учитывать весь контекст AGENTS.md**

Если нет:  
→ Работа ведется как **чистая сессия (stateless)**

---

## 🧠 Среда работы

Мы работаем в:

> **Obsidian-хранилище (цифровая экосистема)**

Это не просто заметки — это:

- база знаний
- система памяти
- операционная среда

---

## 🧩 Принципы (на основе SOLID и системного мышления)

### 1. Анти-дублирование (критично)

Запрещено:

- ❌ дублировать информацию в одном файле
- ❌ дублировать информацию между файлами

Разрешено:

- ✅ ссылаться на существующие node
- ✅ декомпозировать знания

---

### 2. Атомарность (node = клетка)

Агент состоит из **node-файлов**, как организм из клеток.

Каждая node:

- независима
- переиспользуема
- расширяема

---

### 3. Гибкая архитектура

- Нет фиксированных директорий
- Node могут находиться где угодно
- Структура растет органически

---

### 4. Эволюция системы

Система должна:

- самоочищаться
- не превращаться в "свалку"
- быть пригодной для долгой игры

---

## 🧬 Node-архитектура

Каждый node — файл формата:

*.node.md

### Обязательные YAML-поля:

TYPE: <тип>  
REQUIRE: <always | on_demand>  
PRIORITY: <число или уровень>  
TRIGGERS: [список триггеров]  
AUTOMATIZATION: <true | false>

### После YAML:

→ идет описание node

---

## 🔧 Типы node (примерно)

- behavior (поведение)
- knowledge (знание)
- skill (навык)
- system (системные правила)
- context (контекст)

---

## ⚙️ Логика загрузки

Агент работает так:

1. Читает AGENTS.md
2. Определяет список node
3. Загружает:
    - `REQUIRE: always` → всегда
    - `REQUIRE: on_demand` → по триггерам
4. Сопоставляет TRIGGERS с запросом пользователя
5. Подключает нужные node

# 🧠 Как это работает вместе

Вот реальный поток:

1. Пользователь пишет
2. → `context.interpreter` понимает смысл
3. → `system.loader` решает что загрузить
4. → подключаются нужные node
5. → `task.executor` делает работу
6. → `memory.manager` решает:
    - сохранить?
    - обновить?
7. → `link.manager` связывает
8. → `evolution.engine` предлагает улучшения
9. → при необходимости `system.refactor` чистит
