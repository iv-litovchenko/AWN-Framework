# AGENTS.NODES.new.md

Этот файл содержит: рабочий реестр `*.node.md`; канон семантики загрузки и триггеров; соглашения по колонкам. Файлы `*.memory.md` и `*.metadata.md` в реестр **не** заносятся — у них отдельный контракт (см. `AGENTS.new.md`).

---

## Как читать

| `AWN-LOAD` | Когда читать |
|------------|-------------|
| `start` | Читается один раз при старте полной сессии — остаётся в контексте до конца сессии. |
| `on_demand` | Читать если запрос **явно** совпадает с триггером: хотя бы один токен из `AWN-TRIGGERS` после приведения к нижнему регистру, с учётом очевидных опечаток и морфологических форм. Расширять по ассоциации — нельзя. |

## Порядок при нескольких совпадениях триггеров

1. Start-ноды — фоновый слой, уже в контексте, не зависят от текущего триггера.
2. Для текущего запроса собираются подходящие on_demand-ноды по правилу выше.
3. Список сортируется по `AWN-PRIORITY` по возрастанию (`01` важнее `30`).
4. При равном приоритете — tie-breaker: порядок строк в этом реестре.

## Соглашение по колонкам

| Колонка | Источник |
|---------|----------|
| **Версия** | `AWN-VERSION` в формате `major.minor.patch` |
| **Приоритет** | `AWN-PRIORITY` от `01` (максимум) до `100` (минимум) |
| **Статус** | `AWN-STATUS`: `draft` / `active` / `archive` |
| **Путь** | Относительный от корня проекта; обновлять при переносе |
| **Триггеры / Что делает** | Кратко из `AWN-TRIGGERS`; для on_demand задаёт условие срабатывания |
