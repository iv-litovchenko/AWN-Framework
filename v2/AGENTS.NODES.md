# AGENTS.NODES.md

Рабочий реестр `*.node.md` для v2. Файлы `*.memory.md` и `*.metadata.md` в реестр **не** заносятся.

---

## Как читать

| `AWN-LOAD` | Когда читать |
|------------|-------------|
| `start` | Читается один раз при старте полной сессии — остаётся в контексте до конца сессии. |
| `on_demand` | Читать если запрос **явно** совпадает с триггером: хотя бы один токен из `AWN-TRIGGERS` после приведения к нижнему регистру, с учётом очевидных опечаток и морфологических форм. Расширять по ассоциации — нельзя. |

## Порядок при нескольких совпадениях триггеров

1. Start-ноды — фоновый слой, уже в контексте, не зависят от текущего триггера.
2. Для текущего запроса собираются подходящие on_demand-ноды по правилу выше.
3. Список сортируется по `AWN-PRIORITY` по возрастанию (`01` важнее `30`).
4. При равном приоритете — tie-breaker: порядок строк в этом реестре.

## Соглашение по колонкам

| Колонка                   | Источник                                                            |
| ------------------------- | ------------------------------------------------------------------- |
| **Версия**                | `AWN-VERSION` в формате `major.minor.patch`                         |
| **Приоритет**             | `AWN-PRIORITY` от `01` (максимум) до `100` (минимум)                |
| **Статус**                | `AWN-STATUS`: `draft` / `active` / `archive`                        |
| **Путь**                  | Относительный от корня проекта; обновлять при переносе              |
| **Триггеры / Что делает** | Кратко из `AWN-TRIGGERS`; для on_demand задаёт условие срабатывания |

## Области (NODE/AREA)

Справочник областей vault — только для ориентации агента в структуре. Area-ноды **не входят** в таблицы загрузки выше; агент читает их контекстно при работе внутри соответствующей области.

| Статус | Путь                              | Описание                                                                                      |
|--------|-----------------------------------|-----------------------------------------------------------------------------------------------|
| active | `Assistant.Ai/Area.node.md`       | Область ассистента: кто он, кто его пользователь, по каким правилам работает                  |
| disabled | `Assistant.Ai/Core/Area.node.md`  | Системные скрипты и утилиты ассистента (заглушка — наполняется по мере роста)               |
| disabled | `Assistant.Ai/Chats/Area.node.md` | История чатов с агентом: сохранённые сессии в формате `.md`                                 |
| disabled | `Assistant.Ai/Nodes/Area.node.md` | Контент-ноды ассистента: навыки, домены, ситуативные правила                                |
| disabled | `Assistant.Ai/Logs/Area.node.md`  | Логи работы агента: изменения, ошибки, аудит действий                                       |
| active | `Inbox/Area.node.md`              | Входящие материалы: всё неразобранное — ссылки, идеи, черновики; разбираются вместе с агентом |
