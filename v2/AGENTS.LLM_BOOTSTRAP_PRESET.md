# AGENTS.LLM_BOOTSTRAP_PRESET.md

Общий протокол старта для LLM-совместимых агентных сессий (v2).

---

## Bootstrap (обязательно)

Перед первым ответом в каждой новой сессии ты ОБЯЗАН:

1. По **первому** сообщению пользователя определить режим: в **начале** есть приветствие вроде «Привет» или «Hi» — **полная сессия**; иначе — **чистая сессия**.

2. **Полная сессия:**
   - Прочитать `./AGENTS.md`, затем `./AGENTS.NODES.md`.
   - Загрузить все ноды с `AWN-LOAD: start` (таблица «при старте» в реестре) — прочитать их файлы один раз, они остаются в контексте до конца сессии.
   - Ноды с `AWN-LOAD: on_demand` (таблица «по необходимости») на старте **не читать** — обращаться к ним только при явном совпадении запроса с `AWN-TRIGGERS`.
   - Не спрашивать разрешения на чтение этих файлов.

3. **Чистая сессия:** **не** читать `./AGENTS.md` и **не** читать `./AGENTS.NODES.md`; отвечать только по явной задаче и явно приложённым файлам.

4. Только после выполнения шагов по выбранному режиму формировать первый ответ.

5. Если выбрана **полная сессия** и `./AGENTS.md` не найден в текущей директории — сообщить об этом и попросить пользователя указать корректный путь.

6. В стартовом сообщении не перечислять внешние протоколы и не предлагать читать файлы вне текущего проекта.

---

## Источник правил

`AGENTS.md` (в корне текущего проекта) — единый источник системных договорённостей в **полной сессии**: протоколы прозрачности, правила нод, конвенции файлов и всё остальное. В чистой сессии этот файл намеренно не загружается.
