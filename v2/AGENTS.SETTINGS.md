## Предустановленные настройки и правила


- [ ] Всегда использовать стандартный Markdown — без `[[...]]` и `![[...]]`:**

	Для чего -
	
	ссылки, изображения и якоря работали одинаково предсказуемо в Obsidian и в GitHub;
	
	- не попадали в репозиторий wiki-встраивания `[[...]]` и `![[...]]`, которые GitHub не рендерит.

	- Изображения: `![alt](Preview.png)`
	- Ссылки на файлы: `[текст](README.md)`
	- Якоря: `[раздел](README.md#раздел)`
	- Встраивание файлов — не использовать, заменять обычной ссылкой
	- Пути — относительные от текущего файла, без `../`
	
	После этого новые ссылки будут создаваться в markdown-формате (который понимает GitHub).
	
	***В Obsidian работает:***
	
	- `![[Preview.png]]`
	- `[[README#Раздел]]`
	
	***В GitHub работает:***
	
	- `![Preview](Preview.png)`
	- `[ссылка](README.md)`
	- `[раздел](README.md#anchor)` (только в markdown-стиле)
	
	**В Obsidian (настройки):**
	
	Settings → Files and links:
	
	- **Use [[Wikilinks]]** — выключить
	- **New link format** — выбрать `Relative path to file`
