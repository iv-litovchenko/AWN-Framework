## Назначение

Нода с **`AWN-REQUIRE: start`**: в **полной** сессии текст подмешивается **всегда**, чтобы агент стабильно отличал **`*.metadata.md`** от ноды и от **`*.memory.md`**.

Файлы **`{полное_имя_носителя}.metadata.md`** в **той же папке**, что и бинарный носитель (изображение, аудио, видео, PDF и т.д.) — это **sidecar**: текст, который агент читает вместо «немого» файла. Канон имени и краткий ориентир по маскам — в `AGENTS.md` → **«Ориентир для агента»** и короткий блок **«Sidecar метаданные»**; здесь — развёрнутое руководство.

## Чем sidecar не является

- **Не** `*.node.md` — в реестр `AGENTS.NODES.md` не попадает.
- **Не** `*.memory.md` — нет **`AWN-OWNER-NODE`** и **`AWN-TYPE: "MEMORY/RECORD"`**; это не журнал фактов по ноде, а описание **конкретного файла-носителя**.

## Правило имени (канон AWN)

Имя sidecar = **полное имя носителя** + **`.metadata.md`**, каталог совпадает с каталогом носителя.

Примеры: `doc/Aya.jpg` → `doc/Aya.jpg.metadata.md`; `media/talk.m4a` → `media/talk.m4a.metadata.md`.

## Поля в шапке (все опциональны; без префикса `AWN-`)

Ключи **на английском**, префикс **`AWN-`** в sidecar **не** используем — иначе путается с контрактом ноды и памяти ноды.

| Ключ | Зачем |
| ---- | ----- |
| `summary` | Коротко: что за носитель, зачем в vault. Чаще всего достаточно этого (и при желании `TAGS`). |
| `relatedNode` | **Только если нужен контекст:** относительный путь к `*.node.md` (персона, проект, коллекция), к которой **смыслово** относится этот ассет. У случайного скриншота поле **не** обязательно — можно не указывать. |
| `transcript` / `notes` | Расшифровка, таймкоды или продолжение в теле ниже шапки. |
| `mediaKind` | Подсказка: `image`, `audio`, `video`, `document` и т.п. |
| `TAGS` | Поиск по vault. |
| `created` / `updated` | ISO `YYYY-MM-DD` для **описания**; не путать с `AWN-CREATED` / `AWN-UPDATED` у нод и `*.memory.md`. |

Тело после `---` — произвольный markdown (детали, `![[...]]` на носитель, ссылки).

## Минимальный пример (без привязки к ноде)

```yaml
---
summary: "Постер AYA Cognitive Intelligence System — визуал продукта."
TAGS: [aya, branding, poster]
---

Далее в файле — тело: ссылка на носитель `doc/Aya.jpg`, при необходимости развёрнутое описание.
```

## Пример с опциональной привязкой к ноде контекста

Добавляйте **`relatedNode`** только если ассет явно относится к одной ноде (брендбук персоны, обложка проекта и т.д.):

```yaml
---
summary: "Постер AYA — визуал продукта, для контекста персоны."
relatedNode: Nodes/Core/Persona.node.md
mediaKind: image
TAGS: [aya, branding, poster]
created: 2026-04-16
updated: 2026-04-16
---

Далее в файле — тело: подробности, ссылка на `doc/Aya.jpg`.
```

## Плагины Obsidian

Отдельные плагины (companion к медиа) могут использовать похожую идею; **в AWN зафиксированы** только маска **`.metadata.md`** и правило **`{имя_носителя}.metadata.md`**.
