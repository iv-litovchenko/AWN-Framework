# MemPalaceIntegration — Интеграция внешней системы долгосрочной памяти

**Назначение области:**
Использовать MemPalace как **дополнительный слой verbatim-памяти** (полная история разговоров без сокращений) в дополнение к нашей чистой системе nodes + areas.

**Как мы используем MemPalace вместе с нашей системой:**

1. **Наша система (Nodes + Areas)** — остаётся основой  
   - Чистая, структурированная, ручная/курируемая память  
   - No Duplication, принципы, ценности, архитектура

2. **MemPalace** — слой «полной истории»  
   - Хранит **verbatim** (оригинальные сообщения) всей переписки  
   - Делает мощный **семантический поиск** по всему прошлому

**Правила интеграции:**
- Важные insights и решения мы **вручную** переносим из MemPalace в наши `.area.node.md` / `.node.md`
- MemPalace используем только когда нужно найти «что именно мы говорили 3 месяца назад»
- Не дублируем информацию — MemPalace = сырая история, наши nodes = очищенное знание

**Как я обращаюсь к MemPalace:**
- Через MCP-инструменты (если подключено)
- Или через команду `mempalace search "запрос"`
- Или через `mempalace wake-up` (загружает ключевой контекст)

**Когда активировать:**
- При запросах вида «вспомни что мы говорили про…», «найди все упоминания…», «что было в марте»
- При больших review (ежеквартальных)
