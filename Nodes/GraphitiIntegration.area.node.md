# GraphitiIntegration — Интеграция Temporal Knowledge Graph

**Назначение области:**
Использовать **Graphiti** (от Zep AI) как продвинутый третий слой памяти для автоматического построения **временного графа знаний** из наших обсуждений.

### Три слоя Permanent Memory (финальная архитектура)

**Слой 1 — Курируемая память (Obsidian Nodes)**
- `*.area.node.md` + atomic nodes
- Чистые, отредактированные человеком контракты, принципы, ценности
- Главный источник истины (No Duplication)

**Слой 2 — Verbatim-память (полная история)**
- MemPalace / sqlite-vec
- Сырые тексты всех наших чатов
- Семантический поиск по полным обсуждениям

**Слой 3 — Temporal Graph (Graphiti)**
- Автоматически извлекает сущности, отношения и **временную эволюцию** фактов
- Каждый факт знает "когда он был верен" и "когда устарел"
- Полная provenance (ссылка на исходный эпизод чата)

### Как мы будем использовать Graphiti

1. После важного обсуждения я отправляю эпизод в Graphiti.
2. Graphiti автоматически:
   - Извлекает сущности (проекты, цели, правила, ценности)
   - Создаёт/обновляет связи с временными метками
   - Помечает устаревшие факты
3. При запросах я могу спрашивать:
   - «Как изменились наши правила по финансам за последние 4 месяца?»
   - «Какие были разные версии нашего подхода к планированию?»
   - «Покажи эволюцию решения по здоровью»

**Преимущества Graphiti для нас:**
- Отлично решает проблему потери контекста и эволюции решений
- Hybrid retrieval (семантика + граф + время)
- Инкрементальные обновления (не нужно перестраивать граф каждый раз)

**Статус интеграции:** Планируется (апрель–май 2026). Будет работать через MCP-сервер или Python API.

**Технические требования:** Neo4j или FalkorDB + Python сервис.
