# Domains — реестр доменов пользователя

> [!info] О файле
> Нода (`*.node.md`, `NODE/INDEX`): точка входа в папку `Domains/`. Здесь живут: конвенция структуры каждого домена (папки, форматы, правила Notes/Memory/Assets), инструкция по созданию нового домена и реестр всех доменов пользователя.

## Что такое домен

**Домен** — самодостаточная область жизни или работы пользователя: тема, проект, сфера. Каждый домен живёт в своей подпапке и опирается на фиксированную структуру (см. ниже).

## Структура домена (конвенция)

Каждая папка `Domains/ИмяДомена/` может содержать (по необходимости):

```
Domains/ИмяДомена/
  Index.node.md     ← дашборд домена: цели, список тем Notes/, AWN-CATEGORY, правила агента
  Notes/            ← обычные .md заметки по теме домена (создаётся при необходимости)
  Assets/           ← бинарные файлы домена (PDF, картинки, аудио, документы)
  Memory/           ← *.memory.md — записи памяти нод этого домена
  Nodes/            ← опционально: AWN-ноды специфичные для домена
  Dashboards/       ← опционально: доменные дашборды и навигационные представления
  templates/        ← опционально: доменные шаблоны заметок и нод
```

### Соглашения по именам файлов

| Папка     | Формат имени              | Пример                         |
| --------- | ------------------------- | ------------------------------ |
| `Notes/`  | `НазваниеТемы.md`         | `Verbs.md`, `PastTense.md`     |
| `Assets/` | `YYYY-MM-DD-описание.ext` | `2026-05-04-grammar-unit3.pdf` |
| `Memory/` | `YYYY-MM-DD.memory.md`    | `2026-05-04.memory.md`         |

### Что такое Notes, Memory и Assets

**`Notes/`** — справочник и конспекты. Один файл на тему, редактируется и пополняется со временем. Агент не подгружает автоматически — доступны через Dataview-блок в `Index.node.md` или по явному запросу. Шаблон: `awn-templates/Memory/note.template.md`.

> Используй если: тема изучается долго и знания накапливаются в одном месте.
> Пример: `Verbs.md` — конспект по глаголам, который пополняешь по мере изучения.

**`Memory/`** — лог событий и сессий. Один файл на дату, создаётся агентом по запросу. Имеет AWN-контракт (`AWN-OWNER-NODE`). Фиксирует что происходило, а не что ты знаешь.

> Используй если: нужно записать событие, прогресс за день, встречу, результат сессии.
> Пример: `2026-05-04.memory.md` — «изучили Past Simple, выучили 10 слов».

**`Assets/`** — бинарные файлы (PDF, картинки, аудио). Агент читает их через sidecar `*.metadata.md` рядом с каждым файлом.

| Вопрос | Куда |
|--------|------|
| Это справочник / конспект по теме? | `Notes/` |
| Это событие / лог / прогресс с датой? | `Memory/` |
| Это файл (PDF, картинка, аудио)? | `Assets/` |

Ссылка из заметки на ассет: относительный путь `../Assets/имя-файла`.

## Как создать новый домен

1. Создать подпапку `Domains/ИмяДомена/` (CamelCase)
2. Создать `Index.node.md` на основе шаблона `awn-templates/index.template.md`; задать `AWN-CATEGORY` (slug из `categories.node.md`)
3. Создать нужные подпапки по задаче (обычно: `Notes/`, `Assets/`, `Memory/`; опционально: `Nodes/`, `Dashboards/`, `templates/`)
4. Зарегистрировать домен в таблице ниже и в `AGENTS.NODES.md`
