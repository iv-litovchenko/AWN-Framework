# Управление нодами

> [!info] Назначение
> Эта нода содержит правила и инструкции для работы с нодами AWN-Framework: создание, редактирование, регистрация в реестре и синхронизация.

## Создание новой ноды

### Шаг 1: Определить тип ноды

| Тип | Паттерн имени | Когда использовать |
|-----|---------------|-------------------|
| `NODE/SOLO` | `Имя.node.md` | Одиночная нода — правило, навык, контекст |
| `NODE/SPACE` | `_Index.node.md` | Точка сборки области, оркестратор папки |

### Шаг 2: Написать YAML-шапку

Сначала все поля `AWN-*`, затем пользовательские. Пример для `NODE/SOLO`:

```yaml
---
AWN-TYPE: NODE/SOLO
AWN-TITLE: Название ноды
AWN-DESC: "Зачем эта нода — 1-2 строки"
AWN-LOAD: start | on_demand
AWN-PRIORITY: 30
AWN-TRIGGERS: [слово1, слово2]
AWN-STATUS: active | disabled | draft | archive
AWN-CATEGORY: general
AWN-MEMORY: none | internal | external | hybrid
AWN-AUTOMATIZATION: false
AWN-VERSION: 1.0.0
AWN-CREATED: YYYY-MM-DD
AWN-UPDATED: YYYY-MM-DD
TAGS: [тег1, тег2]
---
```

### Шаг 3: Написать тело ноды

- Начать с блока `# Название`
- Добавить `> [!info] Назначение` с кратким описанием
- Описать правила, инструкции или контекст

### Шаг 4: Регистрация в реестре

После создания файла обязательно добавить запись в `Awn.Registry.md` в соответствующий раздел:

- `start` → Реестр нод «при старте»
- `on_demand` → Реестр нод «по необходимости»

Формат строки:
```
| active | priority | Путь к ноде | triggers | Описание |
```

## Редактирование ноды

1. Прочитать файл через `read_file`
2. Внести изменения (YAML или тело)
3. Обновить поле `AWN-UPDATED: YYYY-MM-DD`
4. Если изменились триггеры или приоритет — обновить реестр

## Синхронизация реестра

При массовых изменениях или если реестр кажется неактуальным:

1. Найти все `*.node.md` в хранилище
2. Сверить с `Awn.Registry.md`
3. Добавить отсутствующие, удалить неактуальные
4. Обновить метаданные изменившихся

## Принципы безопасной маршрутизации

> **Важно:** если не удаётся однозначно определить область для файла — **задать уточняющий вопрос** пользователю. Запрещено "угадывать" при неоднозначности (2+ равновероятных места).

## Проверка перед регистрацией

- [ ] Путь относителен от корня проекта (не начинается с `/Users/`)
- [ ] `AWN-STATUS: active` (иначе нода не загрузится)
- [ ] `AWN-TRIGGERS` заполнены осмысленно
- [ ] Файл сохранён в нужном месте
