# Git Push — сохранить и отправить

> [!info] О файле
> Нода для сценария **commit + push**: осознанный выбор файлов, стоп при файле **> 30 МБ**, без `git push --force` в общие ветки без явного указания пользователя.

## Назначение

Провести безопасную выкладку: что изменилось → что включить в коммит (в т.ч. с приоритетом **недавно затронутых в сессии** файлов, если они известны из контекста) → проверка размера → сообщение коммита → pull при необходимости → push.

## Порог размера

**30 МБ** = `30 * 1024 * 1024` байт (**31457280**). Любой выбранный для коммита файл **строго больше** этого порога → **не коммитить и не пушить**; сообщить пользователю списком путей и размеров.

Проверка размера файла:

- Linux: `stat -c%s -- путь`
- macOS: `stat -f%z -- путь`

## Алгоритм для агента

1. **`git status`**. При необходимости **`git diff`** / **`git diff --stat`**.
2. Составить список кандидатов: все изменённые / новые / удалённые из `git status`, которые пользователь может захотеть закоммитить.
3. **Предложить пользователю** согласовать набор путей для коммита. Явно отметить файлы, которые **недавно менялись в этой беседе** (если известны). Не выполнять **`git add .`** без явного согласия пользователя на «всё».
4. **Проверка размера:** для каждого согласованного пути к **файлу** (не каталогу целиком — для каталога проверять каждый добавляемый файл) вычислить размер. Если хотя бы один **> 30 МБ** — вывести предупреждение с путями и размерами и **прервать** сценарий (без `git add` / `commit` / `push` для этого набора), пока пользователь не исключит файл или не сменит политику (LFS и т.д.).
5. **`git add`** только по согласованным путям.
6. **`git commit -m "…"`** — осмысленное сообщение (`feat:` / `fix:` / `docs:` / `chore:` / `refactor:` по ситуации).
7. Перед push: **`git pull --rebase`** или **`git pull`** — как принято в репозитории.
8. **`git push`**. При отклонении remote — не использовать **`--force`** к **`main`** / **`master`** без явного указания пользователя.

## Ограничения

- Не коммитить секреты (`.env`, ключи, токены).
- Не добавлять служебные файлы IDE/ОС без запроса пользователя.

## Краткие команды

```bash
git status
git diff
git add путь/к/файлу
git commit -m "docs: кратко что сделано"
git pull --rebase   # если принято в проекте
git push
```
