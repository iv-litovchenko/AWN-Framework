# awn.settings.md

> Системные настройки поведения агента для этого vault.

**Приоритет применения:**

1. Явная инструкция пользователя в текущем запросе
2. Активная настройка из этого файла

Если настройка не нужна — снять галочку (`[ ]`).

---

## Реестр настроек для агента

### Rendering

- [x] Использовать стандартный Markdown для ссылок и изображений вместо `[[wiki-синтаксиса]]` (совместимо с GitHub).

| Элемент        | Obsidian wiki       | Markdown                     |
| -------------- | ------------------- | ---------------------------- |
| Изображение    | `![[photo.png]]`    | `![фото](photo.png)`         |
| Ссылка на файл | `[[README]]`        | `[текст](README.md)`         |
| Якорь          | `[[README#раздел]]` | `[раздел](README.md#раздел)` |

В Obsidian: Settings → Files and links → выключить **Use [[Wikilinks]]**, формат → `Relative path to file`.
### Naming

> Не влияет на работу пакетных менеджеров (`npm`/`pnpm`/`yarn`/`pip`/другие) и их служебные файлы/каталоги.

- [ ] При создании новых папок использовать `kebab-case`: только `a-z`, `0-9` и `-`; без пробелов, `_`, `camelCase` и `PascalCase`.
- [ ] При создании новых файлов использовать `kebab-case`: в имени файла допускается дополнительная точка `.` между сегментами; каждый сегмент должен быть в `kebab-case` (`a-z`, `0-9`, `-`). Запрещены пробелы, `_`, `camelCase`, `PascalCase`, начальная/конечная точка и `..`. Контрактные суффиксы (`.node.md`, `.memory.md`, `.metadata.md`, `_index.node.md`) сохранять как есть.
- [ ] Все YAML-ключи писать только в `lower_snake_case` (`a-z`, `0-9`, `_`), без пробелов, `camelCase`, `PascalCase`, `kebab-case` и точки в ключах.
- [ ] Служебные ключи фреймворка писать с префиксом `awn_` (например: `awn_type`, `awn_status`, `awn_memory`).
- [ ] В frontmatter сначала размещать все ключи фреймворка `awn_*`, затем пользовательские ключи.
- [ ] Человеко-читаемые названия свойств задавать только на уровне представлений (Bases/Dataview aliases), в YAML хранить стабильные машинные ключи.
- [ ] При работе с содержимым любых файлов использовать пути только относительно корня проекта (workspace) или относительно текущей ноды. Абсолютные пути (`/Users/...`) не использовать.

## Как добавить настройку

1. Добавить строку в реестр в формате `- [x] Описание` (или `- [ ]`, если выключено по умолчанию).
2. Размещать настройку в тематическом блоке (`Rendering` / `Naming` / `Greeting` / `Voice` / `Другой`).
3. Если подходящего блока нет — создать новый тематический блок и добавить настройку в него.
4. Делать описание в 1 строку: что включает настройка и в каком контексте применяется.
5. Если агент не знает, где хранить договоренность (конфигурацию/параметр), по умолчанию зафиксировать её в этом файле.
