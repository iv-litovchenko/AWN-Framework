# Awn.Registry

> Рабочий реестр `*.node.md` vault. Работает как **таблица маршрутов**: входящий запрос сопоставляется с `AWN-TRIGGERS` нод и определяет, какие файлы добавить в контекст агента.
## Реестр нод «при старте». читаются при старте каждой полной сессии (`AWN-LOAD: start`).

| `AWN-STATUS` | `AWN-PRIORITY` | Путь к ноде                  | `AWN-TRIGGERS`                                                    | Краткое описание |
| ------------ | -------------- | ---------------------------- | ----------------------------------------------------------------- | ---------------- |
| active       | 30             | `Assistant.Ai/Agent.node.md` | Идентичность ассистента (агента) — имя, характер, принципы, стиль |                  |
| active       | 30             | `Assistant.Ai/User.node.md`  | Профиль текущего пользователя — имя, цели, предпочтения           |                  |
## Реестр нод «по необходимости». читаются по мере необходимости (`AWN-LOAD: on_demand`).

| AWN-STATUS | AWN-PRIORITY | Путь к ноде                   | AWN-TRIGGERS              | Краткое описание                                                             |
| ---------- | ------------ | ----------------------------- | ------------------------- | ---------------------------------------------------------------------------- |
| active     | 10           | `Assistant.Ai/_Index.node.md` |                           | Область ассистента: кто он, кто его пользователь, по каким правилам работает |
| active     | 40           | `Assistant.Ai/Users.node.md`  | Реестр всех пользователей |                                                                              |
## Реестр нод с автоматизацией для выполнения по расписанию (`AWN-AUTOMATIZATION: true`). 

Сводка для планировщика окружения агента (cron, n8n и т.п.): все ноды, у которых в шапке **`AWN-AUTOMATIZATION: true`**, обязаны быть перечислены здесь. Канон расписания — поле **`AWN-CRON`** в самой ноде.

**Синхронизация (агент):** при создании ноды, при включении/выключении автоматизации, при смене `AWN-CRON` или переносе файла — добавить, обновить или удалить строку в этой таблице; путь как в остальном реестре (от **корня проекта**).

| `AWN-STATUS` | `AWN-CRON` | Путь | `AWN-TRIGGERS` | Краткое описание |
| ------------ | ---------- | ---- | -------------- | ---------------- |
## Реестр дополнительных файлов (читать/обновлять по необходимости)

Справочник важных не-нодовых файлов, которые агент может подключать в контекст по явной задаче пользователя или по рабочей необходимости.

| Файл (путь)          | Описание и назначение                                                                                                                         |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `HEARTBEAT.md`       | Операционный файл состояния/пульса системы: фиксирует текущий статус, контрольные сигналы и полезен для быстрой проверки «живости» процессов. |
| `README.md`          | Главный обзор проекта: концепция, структура, базовые правила и точки входа для онбординга.                                                    |
| `.env`               | Переменные окружения для локального запуска и интеграций; источник runtime-настроек и секретов (читать с осторожностью, не публиковать).      |
| `.gitignore`         | Правила исключения файлов из git: что не должно попадать в индекс и коммиты.                                                                  |
| `docker-compose.yml` | Описание локальной контейнерной инфраструктуры: какие сервисы запускаются вместе и с какими параметрами.                                      |
