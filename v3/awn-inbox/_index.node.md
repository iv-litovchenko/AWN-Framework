# Inbox

> [!info] О ноде
> Единая нода `awn-inbox/`: объединяет философию области и практический процесс обработки входящих. Реестр файлов при разборе формируется по фактическому листингу (`awn-inbox/Assets/`) в текущей сессии.

## Назначение

Inbox — приёмник всего нового и неразобранного. Сюда попадает материал, который ещё не получил место в vault:

- ссылки и статьи для изучения;
- черновые идеи и заметки "на потом";
- файлы и документы без контекста;
- быстрые входящие записи.

Цель Inbox — не хранить, а перерабатывать: после разбора материалы перемещаются в нужные домены или удаляются.

## Правила обработки входящих

Формат упрощён до одного паттерна:

`awn-inbox/Assets/YYYY-MM-DD-описание.<ext>`

Где:

- `YYYY-MM-DD` = дата добавления;
- `описание` = короткое понятное имя через дефис;
- `ext` = реальное расширение файла (`md`, `png`, `pdf`, `mp3` и т.д.).

Если пользователь явно указывает целевое размещение, материал можно сохранять сразу туда, минуя Inbox.

## Процесс разбора Inbox

**Инициация:** пользователь говорит "разберём inbox".

1. Составить реестр: пройти `awn-inbox/`.
2. Для каждого файла предложить действие: переместить в домен / оставить / удалить.
3. Выполнить только подтверждённые пользователем действия.
4. Обновить список разбора и продолжить порциями, если файлов много.

## Ограничения

- Агент не перемещает и не удаляет автоматически без подтверждения пользователя.
- При неоднозначной маршрутизации сначала задаётся уточняющий вопрос.
