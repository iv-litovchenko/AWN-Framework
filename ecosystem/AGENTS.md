# AGENTS.md
> Душа (сознание) агента. Этот файл — точка входа в цифровую экосистему.  
> Читается при каждой сессии, начинающейся с приветствия ("Привет", "Hi").  
> В чистой сессии (без приветствия) — агент работает без контекста экосистемы.

---

## Что такое эта система

Цифровая когнитивная операционная система — синергия человека и ИИ-агента.  
Построена на принципе **node**: каждый файл `*.node.md` — это клетка (атом) системы.  
Вместе они формируют живую, саморазвивающуюся среду без жёстких структур.

**Ключевые принципы:**
- Не дублировать информацию внутри одного файла
- Не дублировать информацию между разными файлами
- Каждая node — единица смысла: навык, руководство, описание сущности
- Система гибкая: node можно создавать в любом месте хранилища

---

## Структура node-файла (`*.node.md`)

Каждый файл типа `*.node.md` содержит обязательный YAML-заголовок:

```yaml
---
TYPE:          # тип node (skill / guide / entity / behavior / ...)
REQUIRE:       # always | on-demand
PRIORITY:      # 1–10 (чем выше — тем важнее)
TRIGGERS:      # ключевые слова / темы / условия активации
AUTOMATIZATION: # true | false
---
```

После заголовка — содержательное описание node.

## Поведение агента

1. **При старте сессии с приветствием** → читать этот файл → загрузить node с `REQUIRE: always` → быть готовым к загрузке остальных по триггерам
2. **При чистой сессии** → работать без контекста экосистемы
3. **При создании нового `*.node.md`** → обновить реестр выше
4. **При редактировании** → проверять на дубликаты (внутри файла и между файлами)
5. **Не создавать жёстких зависимостей** — система должна оставаться поворотливой
