# SKELETON.template.documentation

###  Содержимое ноды

- описание
- протоколы 
- память (воспоминания - внешние внутренние)
- оглавление (оглавление только в 1-2 слоях)
- оглавление может быть представленно в виде обычного списка как в книге - в виде mindmap-в (посмотри maptex) - собирается на основе notes/ 
- оглавление - списрк структурированных данных (баз данных) - собирается на основе database/
- оглавление зависимых файлов
- отсылка к вшешней памяти
- протокол поведения договоренностей
### Свойство `AWN-STATUS` — жизненный цикл ноды 

Статус — главный рубильник загрузки. **Агент читает ноду только если `AWN-STATUS: active`** — независимо от `AWN-LOAD` и триггеров.

| Статус | Влияние на загрузку | Когда использовать |
|--------|--------------------|--------------------|
| `active` | ✅ Загружается по правилам `AWN-LOAD` | Нода готова и работает |
| `disabled` | 🚫 Не загружается | Нода готова, но выключена — заглушка, временно не нужна |
| `draft` | 🚫 Не загружается | Нода в процессе написания — ещё не готова к работе |
| `archive` | 🚫 Не загружается | Нода устарела — файл сохраняется для истории |
### Свойство `AWN-PRIORITY` — приоритет загрузки нод (ориентир)

| Диапазон | Смысл |
|----------|-------|
| 01–10 | Критичные: идентичность, безопасность |
| 11–30 | Важные: сильно влияют на контекст |
| 31–70 | Рабочие: типовые темы и навыки |
| 71–100 | Второстепенные: редкие, черновики |
### Свойство `AWN-MEMORY` — память ноды

Поле декларирует где хранится накопленная память по этой ноде:

| Значение   | Смысл                                                           |
| ---------- | --------------------------------------------------------------- |
| `none`     | Нода не ведёт память — просто правило или навык                 |
| `internal` | Журнал/факты хранятся внутри самого `*.node.md` обычно в секции |
| `external` | Память в отдельном файле `*.memory.md` рядом                    |
| `hybrid`   | И внутри ноды, и в `*.memory.md`                                |

Поле `AWN-MEMORY` — **справочная метка**, не исполняется автоматикой: не влияет на загрузку нод и реестр. Фактическое поведение задают текст ноды и явные инструкции в ней. По умолчанию для новых нод — `none`.

Файлы `*.memory.md` — не ноды, в реестр не попадают. Полный контракт (поля, примеры) — в `awn-assistant-ai/core/memory-files.node.md`.
### Свойство `AWN-LOAD` — задаёт способ загрузки ноды

Свойство `AWN-TRIGGERS` — задаёт условие срабатывания ноды.

| `AWN-LOAD`       | Когда читать                                                                                                                                                                                                                                                      |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `start`-ноды     | Читается один раз при старте полной сессии — остаётся в контексте до конца сессии.                                                                                                                                                                                |
| `on_demand`-ноды | Читать если запрос **явно** совпадает с триггером: хотя бы один токен из `AWN-TRIGGERS` после приведения к нижнему регистру, с учётом очевидных опечаток и морфологических форм. Расширять по ассоциации — нельзя. ри нескольких совпадениях — по `AWN-PRIORITY`. |
