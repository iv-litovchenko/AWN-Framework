### Твоя текущая философия (как я её понял сейчас):

### Предлагаю чистый и минималистичный паттерн:

### Основное правило именования

**Все файлы заканчиваются на .node.md**

Разделение делается только через **префикс/постфикс типа**:

|Тип файла|Название файла|Назначение|
|---|---|---|
|**Шаблон типа** (прототип)|Space.node.md|Определяет, что такое Space в целом|
|**Конкретная нода**|Маркетинг.space.node.md|Конкретный экземпляр пространства "Маркетинг"|
|**Шаблон типа**|Part.node.md|Что такое Part|
|**Конкретная нода**|Аналитика.part.node.md|Конкретная часть "Аналитика"|
|**Шаблон типа**|Page.node.md|Что такое Page|
|**Конкретная нода**|Главная.page.node.md|Конкретная страница|

### Преимущества этого подхода:

- Всё остаётся **нодой** — концепция не ломается.
- По имени файла сразу понятно, какого типа нода.
- Шаблоны типов лежат в +SYSTEM/Templates/
- Экземпляры лежат в рабочих папках.
- Не нужно учить новые слова (class, object, type и т.д.).

### Как это выглядит на практике

**В папке шаблонов:**

- +SYSTEM/Templates/Space.node.md
- +SYSTEM/Templates/Part.node.md
- +SYSTEM/Templates/Memory.node.md

**В рабочих папках:**

- Areas/Маркетинг/Маркетинг.space.node.md
- Areas/Маркетинг/Контент.part.node.md
- Memory/ShortTerm/Текущий-диалог.memory.node.md

Можно даже упростить ещё сильнее для экземпляров:

- Маркетинг.space.node.md → Space.md (внутри папки Маркетинг)
- Но первый вариант лучше для поиска по всему хранилищу.
