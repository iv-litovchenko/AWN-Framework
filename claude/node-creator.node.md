# Как создавать node-файлы

## Когда создавать новый node
- Появляется повторяющийся паттерн в работе → оформи в node
- Нужно зафиксировать правило или навык → node
- Описываешь сущность (проект, человека, концепцию) → node

## Алгоритм создания
1. Придумать минимальное имя: `тема.node.md`
2. Заполнить YAML-шапку (все 5 полей обязательны)
3. Написать содержание — только суть, без воды
4. Добавить строку в реестр `AGENTS.md`
5. Проставить `[[wikilinks]]` если есть связанные файлы

## Выбор TYPE
| TYPE | Когда использовать |
|---|---|
| `identity` | Кто я, как себя вести |
| `guide` | Правила работы с чем-либо |
| `skill` | Конкретный навык или способность |
| `entity` | Описание объекта: проект, человек, место |
| `behavior` | Паттерн поведения в определённой ситуации |
| `meta` | Node о самой системе |

## Выбор REQUIRE
- `always` — загружается при каждой сессии с приветствием (держи таких мало, max 5–7)
- `on-demand` — загружается только когда сработал триггер

## Хороший триггер
- Конкретный: `задачи, планирование, todo` — хорошо
- Размытый: `работа, помощь` — плохо, будет срабатывать на всё
