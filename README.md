# Node-Test: когнитивная среда человека и агента

Этот проект — прототип персональной AI-экосистемы на Markdown-файлах.
Главная идея: агент работает не из "пустоты", а через явные правила и ноды (`*.node.md`), которые описывают знания, поведение и контекст.

## Что здесь главное

- `AGENTS.md` — базовые правила системы (протокол, принципы, границы, формат нод).
- `AGENTS.NODES.md` — реестр нод (рабочий список).
- `AGENTS.NODES.EXAMPLE.md` — примеры нод по разным областям.
- `CLAUDE.md`, `QWEN.md`, `GEMINI.md`, `CODEX.md` — стартовые инструкции для конкретных агентов.
- `.env` — локальные настройки окружения (секреты не публикуются).

## Как это работает

1. Пользователь начинает сессию.
2. Агент читает `AGENTS.md`.
3. Агент читает `AGENTS.NODES.md`.
4. По триггерам выбирает нужные `*.node.md`.
5. Формирует ответ/действие с учетом контекста и правил.
6. Если ноды изменились — обновляет реестр.

## Схема потока

```text
Пользовательский запрос
        |
        v
AGENTS.md (правила)
        |
        v
AGENTS.NODES.md (реестр)
        |
        v
Поиск релевантных node по TRIGGERS
        |
        v
Проверка Area.node.md / Index.node.md в области
        |
        v
Чтение целевой node + формирование ответа
        |
        v
Обновление реестра при изменениях
```

## Как агент использует `*.node.md` (простой алгоритм)

1. Пользователь задал вопрос (или поставил задачу).
2. Агент ищет ноду по триггерам (`TRIGGERS`: ключевые слова/темы).
3. Перед чтением целевой ноды проверяет: есть ли в этой области `Area.node.md` или `Index.node.md`.
4. Если есть — читает их, чтобы понять:
   - контекст раздела (о чем область в целом),
   - особый тон и правила,
   - связи с другими файлами.
5. Затем читает целевую ноду и дает ответ в правильном контексте.

## Протокол обновления нод

При создании/изменении/удалении `*.node.md`:

1. Выполнить поиск всех `*.node.md` по всему хранилищу.
2. Сверить найденные файлы с `AGENTS.NODES.md`.
3. Обновить реестр (добавить новые, обновить измененные, удалить неактуальные).
4. Убедиться, что пути в реестре относительные от корня текущего проекта.
5. При необходимости обновить `AGENTS.NODES.EXAMPLE.md`.

## Старт для новых сессий

Если агент поддерживает стартовые файлы (`CLAUDE.md`, `QWEN.md`, `GEMINI.md`, `CODEX.md`), он должен:

1. Читать только `./AGENTS.md` из текущего проекта.
2. Не выходить в родительские директории без разрешения.
3. Работать в контексте текущего vault/проекта.
