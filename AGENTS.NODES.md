# AGENTS.NODES.md — реестр и карта node-файлов

Этот файл содержит:

- рабочий реестр `*.node.md`;
- **единый канон** семантики `AWN-REQUIRE` / колонки «Загрузка», триггеров и **порядка** при нескольких совпадениях (разделы ниже); в `AGENTS.md` на это только отсылка;
- соглашения по колонкам реестра.

Колонки реестра заполняются **только** из канонических ключей `AWN-*` в шапке узла; прочие (пользовательские) поля frontmatter на реестр не влияют, но могут читаться агентом вместе с телом ноды.

## Реестр узлов (`*.node.md`)

Заполняйте строки по мере появления файлов.

### Узлы с загрузкой «при старте» (`AWN-REQUIRE: start`)

| Версия | Статус | Путь к файлу | Загрузка | Приоритет | Автоматизация | Триггер (тема или описание) | Создано |
| ------ | ------ | ------------ | -------- | --------- | ------------- | --------------------------- | ------- |
| —      | —      | —            | —        | —         | —             | —                           | —       |

### Узлы с загрузкой «по необходимости» (`AWN-REQUIRE: on_demand`)

Строка, где в **«Путь к файлу»** указано **`—`**, — не узел, а **памятка** по колонке «Триггер» для всех строк с загрузкой **«по необходимости»**; файл по такой строке не открывать.

| Версия  | Статус | Путь к файлу                   | Загрузка         | Приоритет | Автоматизация | Триггер (тема или описание)       | Создано    |
| ------- | ------ | ------------------------------ | ---------------- | --------- | ------------- | --------------------------------- | ---------- |
| `1.0.0` | active | `Nodes/RandomAnecdote.node.md` | по необходимости | 30        | false         | анекдот, случайный анекдот, шутка | 2026-04-15 |

Примеры нод вынесены в отдельный файл: **`AGENTS.NODES.EXAMPLE.md`**.

## Как читать файл по значению «Загрузка»

Здесь зафиксировано, **когда** агент открывает файл узла. Повторять ту же логику в `AGENTS.md` не нужно.

| Значение в таблице / `AWN-REQUIRE` в YAML | Как читать узел |
| ------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **При старте** (`start`)              | Содержимое файла **читается при старте сессии** (в т.ч. при «полной сессии» по приветствию) **автоматически и обязательно** — узел входит в базовый контекст.                                                                                                                                                                                                                                                                  |
| **По необходимости** (`on_demand`)    | Содержимое файла читаем **только если** запрос по смыслу попадает под триггеры. Базовое правило: после приведения к нижнему регистру и разбиения на слова должен совпасть **хотя бы один токен** из `AWN-TRIGGERS` или колонки «Триггер» (включая их простые варианты с **очевидными опечатками** вроде «анегдот» ↔ «анекдот»). Расширять совпадение за счёт произвольных ассоциаций и «похожих тем» **нельзя** — только явно перечисленные слова и их очевидные опечатки/морфологические формы. Не подмешивай ноду без явной тематической связи. |

## Порядок при нескольких совпадениях триггеров

Если по одному запросу подходят **несколько** `on_demand`-нод (или одновременно учитываются и фоновые `start`):

1. Ноды с **`AWN-REQUIRE: start`** уже входят в контекст **полной** сессии по правилам старта — это **фоновый слой**, не зависящий от текущего триггера.
2. Для **текущего запроса** отдельно собираются подходящие `on_demand`-ноды по правилам из таблицы «Как читать…» выше.
3. Список этих `on_demand`-нод **сортируется по `AWN-PRIORITY` по возрастанию числа** (`01` важнее, чем `30`; `30` важнее, чем `80`). При **равном** `AWN-PRIORITY` используется **порядок строк в реестре** `AGENTS.NODES.md` (как стабильный tie‑breaker).
4. Чтение идёт **от более приоритетных к менее приоритетным**, не в произвольном порядке. Сколько нод из списка реально подмешать в контекст **за один ответ** (топ‑1, топ‑3 и т.д.) задаёт **интеграция/оболочка** или настройки (`AGENTS.SETTINGS.md`, `.env`), а не этот файл; по умолчанию AWN не фиксирует жёсткое K, важен только **детерминированный порядок**.

## Соглашение по колонкам

- **Версия** — из YAML `AWN-VERSION` в формате `major.minor.patch` (например, `1.0.0`).
- **Путь к файлу** — относительный путь от корня текущего проекта; обновлять при переносе файла.
- **Статус** — из YAML `AWN-STATUS`: `draft`, `active`, `archive`.
- **Загрузка** — из `AWN-REQUIRE`: **`start`** → в таблице пишем **«при старте»**; **`on_demand`** → **«по необходимости»**.
- **Приоритет** — из `AWN-PRIORITY` по шкале `01..100`: `01` = максимум приоритета, `100` = минимум; при нескольких подходящих `on_demand`-нодах порядок применения — см. раздел **«Порядок при нескольких совпадениях триггеров»** выше.
- **Автоматизация** — из YAML `AWN-AUTOMATIZATION` (`да` / `нет` или `true` / `false`).
- **Триггер** — кратко из `AWN-TRIGGERS` или своими словами для людей; для режима **по необходимости** задаёт условие срабатывания. Строка-памятка с путём **`—`** в той же колонке может дублировать общее правило для всех `on_demand` (см. таблицу «Как читать…»).
- **Создано** — дублирует `AWN-CREATED` из шапки узла.
