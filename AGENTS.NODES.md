# AGENTS.NODES.md — реестр и карта node-файлов

Этот файл содержит:
- рабочий реестр `*.node.md`;
- правила чтения по режимам загрузки;
- соглашения по колонкам реестра.

## Реестр узлов (`*.node.md`)

Заполняйте строки по мере появления файлов. Колонки:

| Путь к файлу                                    | Статус | Загрузка | Приоритет | Автоматизация | Триггер (тема или описание) | Создано | Обновлено |
| ----------------------------------------------- | ------ | -------- | --------- | ------------- | --------------------------- | ------- | --------- |
| `Nodes/RandomAnecdote.node.md`                  | active | по необходимости | 30 | false | анекдот, случайный анекдот, шутка | 2026-04-15 | 2026-04-15 |

Примеры нод вынесены в отдельный файл: **`AGENTS.NODES.EXAMPLE.md`**.

## Как читать файл по значению «Загрузка»

| Значение в таблице / `REQUIRE` в YAML | Как читать узел |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **При старте** (`start`)              | Содержимое файла **читается при старте сессии** (в т.ч. при «полной сессии» по приветствию) **автоматически и обязательно** — узел входит в базовый контекст.         |
| **По необходимости** (`on_demand`)    | Содержимое файла читаем **только если** есть **совпадение триггера** с темой/сутью запроса (из `TRIGGERS` и колонки «Триггер»); иначе не подмешиваем без явной нужды. |

## Соглашение по колонкам

- **Путь к файлу** — относительный путь от корня текущего проекта; обновлять при переносе файла.
- **Статус** — из YAML `STATUS`: `draft`, `active`, `archive`.
- **Загрузка** — из `REQUIRE`: **`start`** → в таблице пишем **«при старте»**; **`on_demand`** → **«по необходимости»**.
- **Приоритет** — из `PRIORITET` по шкале `01..100`: `01` = максимум приоритета, `100` = минимум; более приоритетные ноды учитываются раньше и сильнее.
- **Автоматизация** — из YAML `AUTOMATIZATION` (`да` / `нет` или `true` / `false`).
- **Триггер** — кратко из `TRIGGERS` или своими словами для людей; для режима **по необходимости** задаёт условие срабатывания.
- **Создано** / **Обновлено** — дублируют `CREATED` / `UPDATED` из шапки узла.
