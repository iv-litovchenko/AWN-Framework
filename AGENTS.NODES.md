# AGENTS.NODES.md

Рабочий реестр `*.node.md` vault. Работает как **таблица маршрутов**: входящий запрос сопоставляется с `AWN-TRIGGERS` нод и определяет, какие файлы добавить в контекст агента. Файлы `*.memory.md` и `*.metadata.md` в реестр **не** заносятся.

---

## Как читать

| `AWN-LOAD` | Когда читать |
|------------|-------------|
| `start` | Читается один раз при старте полной сессии — остаётся в контексте до конца сессии. |
| `on_demand` | Читать если запрос **явно** совпадает с триггером: хотя бы один токен из `AWN-TRIGGERS` после приведения к нижнему регистру, с учётом очевидных опечаток и морфологических форм. Расширять по ассоциации — нельзя. |

## Порядок при нескольких совпадениях триггеров

1. Start-ноды — фоновый слой, уже в контексте, не зависят от текущего триггера.
2. Для текущего запроса собираются подходящие on_demand-ноды по правилу выше.
3. Список сортируется по `AWN-PRIORITY` по возрастанию (`01` важнее `30`).
4. При равном приоритете — tie-breaker: порядок строк в этом реестре.

## Соглашение по колонкам

| Колонка                   | Источник                                                            |
| ------------------------- | ------------------------------------------------------------------- |
| **Версия**                | `AWN-VERSION` в формате `major.minor.patch`                         |
| **Приоритет**             | `AWN-PRIORITY` от `01` (максимум) до `100` (минимум)                |
| **Статус**                | `AWN-STATUS`: `active` / `disabled` / `draft` / `archive`           |
| **Путь**                  | Относительный от корня проекта; обновлять при переносе              |
| **Триггеры / Что делает** | Кратко из `AWN-TRIGGERS`; для on_demand задаёт условие срабатывания |

## Ноды с расписанием (`AWN-AUTOMATIZATION: true`)

Сводка для планировщика окружения агента (cron, n8n и т.п.): все ноды, у которых в шапке **`AWN-AUTOMATIZATION: true`**, обязаны быть перечислены здесь. Канон расписания — поле **`AWN-CRON`** в самой ноде.

**Синхронизация (агент):** при создании ноды, при включении/выключении автоматизации, при смене `AWN-CRON` или переносе файла — добавить, обновить или удалить строку в этой таблице; путь как в остальном реестре (от **корня проекта**).

| `AWN-CRON` | Статус | Путь | Назначение |
|------------|--------|------|------------|
| `30 8 * * *` | active | `Assistant.Ai/Cron/DailyHealthCheck.node.md` | Ежедневная проверка состояния системы → `Assistant.Ai/Cron/Memory/*.memory.md` (см. тело ноды) |

## Домены пользователя

Пользовательские домены — жизненные и рабочие области. Живут в папке `Domains/`. Каждый домен = подпапка с `Index.node.md` и полем `AWN-CATEGORY`.

Реестр доменов — в **`Domains/Index.node.md`** (единственный источник истины). Справочник категорий — в `Domains/Categories.node.md`.
