# 🗂️ Память: Журнал событий

## 🎯 Назначение
Фиксировать ключевые решения, инсайты и вехи пользователя, чтобы агент мог ссылаться на них в будущем без дублирования в других узлах.

## 📋 Формат записи
```markdown
### {{date}} — [Краткий заголовок]
- **Контекст**: [о чём шла речь]
- **Решение/Инсайт**: [суть]
- **Связанные узлы**: [[ссылка]]
- **Статус**: [активно / на пересмотр / архив]
```

## 🧭 Правила

1. Записывать только то, что пользователь явно попросил запомнить или что является объективной вехой (90 дней привычки, крупная трата, принятое решение).
2. Не дублировать: если тема уже раскрыта в другом узле — поставить ссылку, а не копировать.
3. Раз в месяц предлагать ревизию: «Какие записи ещё актуальны?»
