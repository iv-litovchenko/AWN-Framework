Разберу пошагово, как воссоздать в Obsidian систему памяти, похожую на Д. Ж. А. Р. В. И. С.

## Шаг 1. Настройка хранилища (vault)

Создайте структурированное хранилище с папками:

- `00-INBOX` — для быстрых заметок без сортировки;
    
- `01-CAPTURES` — наблюдения, идеи, вопросы (сортировка по типу, а не теме);
    
- `02-CONNECTIONS` — найденные связи между идеями;
    
- `03-BRIEFS` — готовые брифы для проектов;
    
- `04-PUBLISHED` — завершённые работы;
    
- `05-CLAUDE` — папка для ИИ‑взаимодействия с файлом `CLAUDE.md` и подпапками `skills` и `context`.
    

## Шаг 2. Установка плагинов

Обязательные плагины:

- **Templater** — для шаблонов заметок;
    
- **Dataview** — запросы к базе данных;
    
- **QuickAdd** — быстрый захват информации;
    
- **Obsidian Git** — автоматическое резервное копирование на GitHub.

## Шаг 4. Создание файла CLAUDE.md

Это «ядро» вашей системы. В нём укажите:

- описание вашей личности и целевой аудитории;
    
- основные темы и «столпы» контента;
    
- структуру хранилища (vault);
    
- стиль общения и письма («голос»);
    
- жёсткие правила (например, «не трогать .env», «не менять PUBLISHED без команды»);
    
- список задач для ИИ: обработка INBOX, поиск связей, генерация брифов, написание текстов, ведение лога эффективности.
    

## Шаг 5. Настройка ключевых «скиллов» (навыков)

NaN. **Process Inbox** — обрабатывает сырые заметки из INBOX:
    
    - сокращает до одной ключевой фразы;
        
    - добавляет 3 релевантных тега;
        
    - распределяет по соответствующим папкам.
        
NaN. **Weekly Connections** — еженедельно ищет 4 типа связей:
    
    - один принцип в разных областях;
        
    - противоречия между заметками;
        
    - паттерны из 3+ заметок;
        
    - вопросы из одной заметки, на которые отвечают другие.
        
NaN. **Generate Brief** — создаёт бриф из 5 полей:
    
    - одна центральная идея;
        
    - доказательство/пример;
        
    - трансформация для читателя;
        
    - 3 хука (зацепки);
        
    - 3 варианта завершения.
        
NaN. **Write Content** — пишет текст по структуре:
    
    - hook (зацепка);
        
    - proof (доказательство);
        
    - body (основная часть);
        
    - closer (завершение).

## Шаг 7. Еженедельные и ежемесячные процедуры

**По воскресеньям:**

- запустите **Weekly Connections** на весь массив за 7 дней;
    
- прогоните 2 лучшие связи через **Generate Brief**.
    

**Раз в месяц:**

- проанализируйте папку `PUBLISHED`:
    
    - какие темы дают больше сохранений на показ;
        
    - какие хуки обгоняют среднее;
        
    - какие новые углы подсказаны лучшими постами.

## Что это даёт

Через 30 дней: пропадает состояние «не о чем писать».  
Через 90 дней: тексты становятся конкретнее и оригинальнее.  
Через полгода: работает полноценный «второй мозг» — сотни наблюдений, неочевидных связей и контента на основе вашего мышления.

## Важные нюансы

- **Сортировка по типу, а не по теме** в `CAPTURES` позволяет ИИ находить неочевидные связи между разными областями.
    
- **Чёткие инструкции в CLAUDE.md** критически важны — чем конкретнее, тем лучше работает система.
    
- **Регулярность** — ключ к успеху: ежедневные и еженедельные ритуалы превращают систему из таск‑менеджера в «систему разведки».
